Partners reviewed both WordPress and Payload CMS implementation specs and chose **Payload CMS 3.x + Next.js + PostgreSQL** deployed on Vercel.

**Why:** Decision made after consulting with project partners during downtime between Feb–Mar 2026. Both specs were prepared; Payload was selected.

**How to apply:** All implementation work should follow `specification/Implementation_payload.md`. The WordPress spec (`Implementation_wordpress.md`) is archived for reference only. Memory file `MEMORY.md` architecture section should reflect Payload, not WordPress.
